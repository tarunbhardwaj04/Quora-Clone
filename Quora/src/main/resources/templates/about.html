The project follows a Layered Architecture to ensure clean code, maintainability, and easy testing: Controller Layer: Handles incoming HTTP requests and maps them to service methods. Service Layer: Contains the core business logic, validations, and orchestration between different components. Repository Layer: Utilizes Spring Data JPA for efficient interaction with the MySQL database. Entity Layer: Defines the database schema using Java objects (User, Question, Answer, Topic). DTO (Data Transfer Object): Decouples the internal database structure from the API response to maintain security and flexibility.

ğŸ—„ï¸ Database Management & Migration

Unlike traditional projects that use ddl-auto=update, this project utilizes Flyway for database versioning. Version Control: All schema changes (tables, indexes, constraints) are tracked in src/main/resources/db/migration. Reliability: Hibernate is set to validate mode, ensuring the Java code and Database schema stay perfectly synchronized. Audit Trail: Flyway maintains a flyway_schema_history table to track which migrations have been applied.
